# Влияние трансформаций на качество глобальных моделей прогнозирования временных рядов

## 1. Постановка задачи
**Цель исследования:** Определить, какие математические трансформации временного ряда наиболее эффективны для различных типов данных при использовании глобальных моделей машинного обучения (на примере CatBoost). 

**Задачи эксперимента:**
1. Выполнить кластеризацию выбранного датасета временных рядов для выделения типичных паттернов (трендовые, сезонные, зашумленные и т.д.)
2. Обучить глобальную ML-модель (CatBoost) и сравнить её с классическими бейзлайнами (Naive, Seasonal Naive, AutoETS, AutoTheta).
3. Провести сравнительный анализ качества прогноза глобальной модели на:
   * Исходных данных (Original target);
   * Данных после (`log1p`);
   * Данных после (`box-cox`);
   * Данных после (`differencing`).
4. Сделать выводы о применимости трансформаций для выделенных кластеров.

## 2. Описание данных и методология

### 2.1. Данные
Для эксперимента использовалась случайная подвыборка из **200 временных рядов** датасета **M4 (Monthly)**. Для обеспечения корректного обучения сезонных признаков и построения лагов были отобраны только ряды длиной не менее 60 месяцев. Горизонт прогнозирования составил **18 шагов** (1,5 года).

### 2.2. Кластеризация
Ряды были разделены на группы по стат признакам (tsfeatures: среднее, дисперсия, коэффициент вариации, асимметрия, эксцесс, сила тренда) с последующей кластеризацией. 
Было выделено **4 кластера** по силуэту :
* **Кластер 0 (138 рядов):** Основная масса рядов. Предположительно, ряды с выраженным положительным трендом и умеренной дисперсией.
* **Кластер 1 (40 рядов):** Ряды с высокой дисперсией (ошибка MAE по кластеру значительно выше средней).
* **Кластер 2 (19 рядов):** Ряды со специфической структурой, где дифференцирование показало экстремальное снижение ошибки.
* **Кластер 3 (3 ряда):** Аномальные выбросы / шумовые ряды.

### 2.3. Модели и Признаки
* **Бейзлайны:** Naive, Seasonal Naive, Mean, Median.
* **Статистические авто-модели:** AutoTheta, AutoETS.
* **Глобальная модель:** `CatBoostRegressor` (глубина 6, L2-регуляризация 5.0).
* **Признаки для ML:** Авторегрессионные лаги (1-12), скользящие окна (3, 6, 12), синусоидальные признаки сезонности (Fourier), номер шага, а также мета-признаки (`meta_cluster`, `month_in_period` как категориальные). Использовалась стратегия **Direct** (18 независимых моделей для каждого шага горизонта), чтобы избежать накопления ошибки.

## 3. Результаты экспериментов по сравнению моделей с бейзлайнами

Оценка производилась по метрикам MAE, RMSE, MAPE, sMAPE и MASE. Ключевой метрикой сравнения выбран **MASE** (Mean Absolute Scaled Error), так как он позволяет корректно усреднять ошибку по рядам разного масштаба.

| Трансформация | Модель | MAE | sMAPE (%) | MASE |
| :--- | :--- | :--- | :--- | :--- |
| **None** | Naive | 602.95 | 13.53 | 1.22 |
| **None** | AutoETS | 513.24 | **11.83** | **0.92** |
| **None** | CatBoost | 670.46 | 14.77 | 1.48 |
| **Log1p** | CatBoost | 635.93 | 14.20 | 1.44 |
| **Box-Cox** | CatBoost | 1189.20 | 36.20 | 4.02 |
| **Diff** | CatBoost | **516.63** | 12.36 | 1.00 |

Без трансформаций CatBoost уступает даже простому наивному прогнозу (MASE 1.48 против 1.22), так как модель тяжела для малого набора семплов и не имеет преимуществ глобальной модели. Однако применение взятия разностей (`diff`) меняет ситуацию: CatBoost обходит Naive (MASE 1.00) и приближается к эталонному AutoETS.

## 4. Анализ влияния трансформаций по кластерам

### 4.1. Трансформация: Differencing (Разности)
* **Принцип:** Модель предсказывает не абсолютное значение, а $y_{t+1} - y_t$.
* **Результат:** Победитель для Кластеров 0, 1 и 2.
* **Для каких рядов полезна:** Для рядов с нестационарным средним (ярко выраженный, меняющийся тренд / случайное блуждание). 
* **Почему помогло:** Взятие разностей "привязывает" предсказание ML-модели к последней известной фактической точке (свойство, за счет которого выигрывает Naive). Это решает проблему "якоря" (level shift), когда глобальная модель на длинном горизонте теряет текущий уровень ряда. 

### 4.2. Трансформация: Log1p
* **Принцип:** Сжимает масштаб ряда, применяется как $\ln(1 + y)$.
* **Результат:** Показала небольшое улучшение относительно отсутствия трансформации (MASE снизился с 1.48 до 1.44), но сильно уступила `diff`. 
* **Для каких рядов полезна:** Для рядов с **гетероскедастичностью**, где амплитуда сезонности или дисперсия шума растет пропорционально уровню тренда (мультипликативная сезонность).
* **Почему не стала лучшей:** Большинство рядов в случайной выборке M4-Monthly, вероятно, имеют аддитивный характер, либо тренд доминирует над изменением дисперсии. 

### 4.3. Трансформация: Box-Cox
* **Принцип::** Параметрическое степенное преобразование для стабилизации дисперсии.
* **Результат:** Чудовищное падение качества.
* **Почему это произошло:** Box-Cox крайне чувствителен к подбору параметра $\lambda$. На относительно коротких и зашумленных рядах (какими являются многие месячные выборки в 60-100 точек) оценка $\lambda$ может быть нестабильной. При обратном преобразовании (которое включает экспоненцирование) даже минимальная ошибка модели CatBoost приводила к колоссальным выбросам.

### 4.4. Отсутствие трансформации (None)
* **Результат:** Победила только в Кластере 3 (аномальном, состоящем из 3 рядов).
* **Для каких рядов полезна:** Для полностью стационарных рядов, которые колеблются вокруг неизменной константы без тренда и изменения дисперсии. В реальных экономических (Monthly) данных такие ряды встречаются редко.

## 5. Итоговые выводы

1. **Необходимость трансформаций для ML:** Использование «сырых» данных для глобальных моделей на основе деревьев решений (градиентный бустинг) в задачах прогнозирования рядов неэффективно. Модель не может экстраполировать тренд. Трансформации строго обязательны.
2. **Абсолютное превосходство Diff:** Дифференцирование ряда является наиболее универсальной и мощной трансформацией для CatBoost. Оно делает ряды более стационарными (убирает тренд), позволяя деревьям легко находить закономерности в приращениях.
3. **Опасность параметрических трансформаций:** Сложные трансформации (Box-Cox) без жесткого контроля выбросов на этапе инференса могут разрушить предсказательную силу модели из-за эффекта обратного масштабирования ошибок.
4. **Статистика vs ML на малых данных:** Несмотря на успешное применение `diff` и обход наивного метода, классическая локальная статистика (AutoETS с MASE 0.92) всё ещё превосходит глобальный ML на выборке в 200 рядов. Для полного раскрытия потенциала CatBoost требуется увеличение обучающей выборки (от 1000 рядов и более), чтобы модель смогла выучить сложные кросс-серийные сезонные паттерны.
